HTML

  • The head element contains information about the webpage
  • The body element represents the visible content shown to the user
  • The title in the head element is visible on the browser tab
  • HTML builds the framework for the webpage

CSS

  • CSS styles the website
  • CSS defines the colors, aesthetics, visual layout, and organization
  • css is most commonly done in an external file and linked to the code's head meta data
  • A margin indicates how much space we want around the outside of an element
  • A padding indicates how much space we want around the content inside an element
  • Using class "cards" will help design multiple similar elements at once

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git pull origin main: pulls down the latest main branch
  • git add -A: adds all changed files to the staging area of Git
  • git commit -m "commit message here": commits changes to repository
  • git push origin "branch name" pushes changes in a feature branch to the main branch

JavaScript

  • JS sets up how users interact with the website
  • Java reads top to bottom. Order MATTERS
  • The first item of any array has an index of 0, not 1
  • Indices will always be whole numbers that increase by 1 each time
  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • Functions must be called
